[복습 및 개념 정리]

1. React란?
사용자 인터페이스(UI)를 구축하기 위한 자바스크립트 라이브러리
데이터가 변경될 때 react는 컴포넌트를 효율적으로 업데이트하고 다시 렌더링함
jsx 언어를 사용하여 react의 구조를 보다 쉽게 잡음

- 선언형 
상호작용이 많은 UI를 만들 때 생기는 어려움을 줄여줌

- 컴포넌트 기반
캡슐화된 컴포넌트로 복잡한 UI 만들기 쉬워짐
자바스크립트로 작성되어 다양한 형식의 데이터를 앱 안에서 손 쉽게 전달

- 어디서나 사용가능
기존 코드를 다시 작성하지 않고도 개발 가능

2. 컴포넌트

    (1) 간단한 컴포넌트
        - 데이터를 입력받아 화면에 표시할 내용을 반환하는 역할을 하는 render()라는 메서드를 구현
        - 컴포넌트로 전달된 데이터는 render() 안에서 this.props를 통해 접근 가능
    
    (2) 상태를 가지는 컴포넌트
        - this.state로 접근해섯 내부적인 상태 데이터를 가질 수 있음
        - 컴포넌트의 상태 데이터가 바뀌면 render()가 다시 호출되어 마크업이 갱신됨

    (3) 외부 플러그인을 사용하는 컴포넌트
        - react는 유연해서 다른 라이브러리나 프레임워크를 함께 활용할 수 있음
